# Listed Company Explorer

Auto-refreshing dashboard for NSE-listed Indian companies. Filter by sector and
city, compare companies, drill into financials. ~2,000 companies covered, fully
automatic — no manual maintenance.

## Architecture

```
                 ┌─────────────────┐
   Daily 7am IST │ GitHub Actions  │  fetches NSE list + Yahoo Finance data
                 │ refresh_data.py │  writes data/companies_enriched.csv
                 └────────┬────────┘
                          │ git push
                          ▼
                 ┌─────────────────┐
                 │  GitHub repo    │  source of truth
                 └────────┬────────┘
                          │ auto-redeploy
                          ▼
                 ┌─────────────────┐
                 │ Streamlit Cloud │  serves the UI at your public URL
                 │     app.py      │
                 └─────────────────┘
                          ▲
                          │ HTTPS
                  ┌───────┴───────┐
              any laptop / phone / browser
```

You don't touch anything after one-time setup. New IPOs, delistings, financial
updates — all flow through automatically each morning.

## Setup (one-time, ~15 minutes)

See chat for step-by-step instructions. High-level:

1. Push this repo to GitHub
2. Settings → Actions → enable read/write permissions
3. Actions tab → "Refresh company data" → Run workflow (first run, ~20-30 min)
4. Deploy to Streamlit Community Cloud, point at `app.py`
5. Bookmark the URL

After step 5, the workflow runs every morning at 7 AM IST. Your bookmarked URL
always shows fresh data.

## Files

- `app.py` — Streamlit UI
- `data.py` — data loading + live yfinance helpers for detail view
- `scripts/refresh_data.py` — the daily refresh script
- `.github/workflows/refresh.yml` — GitHub Actions schedule
- `companies_seed.csv` — small fallback list (used only before first refresh runs)
- `data/companies_enriched.csv` — auto-generated by the workflow (~2,000 cos)

## Customization

- Change refresh schedule: edit `cron` in `.github/workflows/refresh.yml`
- Use different data source: rewrite `scripts/refresh_data.py` to pull from
  Screener.in or Tijori or any other source; output schema must match what
  `app.py` reads
- Add fields (margin, working capital days, etc.): add column in
  `refresh_data.py`, display it in `app.py`
